작성 시 유의 사항
1. 코드 수정 시 기존 기능과 디자인이 망가지면 안 됨
2. 백엔드와 AI 연동 부분도 망가지면 안 됨
3. 현재까지 기억한 전체 사항:
4. reference 폴더의 md 자료를 기반으로 코드 작성
5. 명시적 요청이 없으면 코드 작성하지 않음
6. 주석은 한글로 상세하게 작성
7. frontend 폴더 내 코드만 추가/수정/삭제
8. 백엔드/AI 연동 부분은 주석으로 상세히 표시
9. frontend/README.md를 매번 업데이트 (흐름 및 핵심만)
10. 불필요한 md 폴더 생성 금지
11. 현재 브랜치에서는 user 화면과 관련된 작업만 수행
12. 코드 수정 시 기존 기능과 디자인이 망가지면 안 됨
13. 백엔드와 AI 연동 부분도 망가지면 안 됨
14. 수정 시 기존 기능, 디자인, 백엔드/AI 연동을 유지
15. 명시적 요청이 없으면 코드 작성하지 않음
16. Mork 기능은 연동 이전까지는 유지 (추후 제거하라고 명령할 때까지는 유지)
